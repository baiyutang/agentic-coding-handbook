The [Guides](/guides/) cover individual skills. Workflows put them together into repeatable
recipes for common kinds of work — the sequences experienced practitioners reach for again
and again.

## Workflows

- **[Test-Driven Development](/workflows/tdd)** — Turn a spec into a failing test, then let
  the agent make it pass. The workflow agents are unusually good at.
- **[Code Review](/workflows/code-review)** — Use an agent as a first-pass reviewer, and
  review the agent's own output effectively.
- **[Large Refactors](/workflows/refactoring)** — Move safely through big, mechanical
  changes without breaking everything at once.
