# 迈向 10x 工程师

Agentic coding 不是靠让你打字更快来实现 10x 的。它倍增你的方式，是改变**你把注意力花在什么上** —— 从写每一行代码，转向指挥、验证和决策。这一页把整本手册浓缩成真正能产生复利的习惯。

## 心智的转变

倍增来自角色的转变，而非速度的提升：

- **3x** —— 你把 AI 用作自动补全和代码片段。有帮助，但每一行仍由你驱动。
- **10x** —— 你**委派整个任务**：规划、让 agent 实现并验证、审查 diff。你以审查者和架构师的身份运作。
- **20x** —— 你**并行地推进工作**并信任你的验证：多个任务同时在跑，每个都由测试和审查把关，配上清晰的指令和强项目上下文。

这一跃升靠的不是更多工具 —— 而是足够信任一个有纪律的循环，从而敢于放开键盘。

## 清单

逐条过一遍；每条都链到深入讲解的章节。

**为杠杆做好准备**
- [ ] 选对适合你工作方式的工具 —— [工具对比](/zh/stack/tools)
- [ ] 为任务选对模型，而非反射性地选最大的 —— [模型选择](/zh/stack/models)
- [ ] 一份精简、高信号的 [`CLAUDE.md`](/zh/reference/claude-md)，让你永不重复解释技术栈

**指挥，而非口述**
- [ ] [具体的提示](/zh/guides/prompting)：范围、约束、示例，以及*为什么*
- [ ] 对任何非琐碎的事[先规划再动手](/zh/guides/planning)；审查规划，而不是 400 行代码
- [ ] 把工作拆成小的、[可审查的切片](/zh/guides/planning)

**信任循环**
- [ ] 给 agent 一个[客观信号](/zh/guides/verification) —— 测试、类型、构建
- [ ] 坚持让它自己运行检查并修复失败；"应该能过"不等于"过了"
- [ ] 在测试能定义"完成"的地方优先用 [TDD](/zh/workflows/tdd)

**像回事一样地审查**
- [ ] 读 [diff，而不是那段描述](/zh/workflows/code-review)
- [ ] 把注意力放在意图、设计和风险上 —— 机械性的事交给 agent
- [ ] 盯住范围：抓出那些你没要求的"顺手"改动

**保持精简**
- [ ] 一个任务一个会话；用 [`/clear` 和 `/compact`](/zh/guides/context-management) 对抗漂移
- [ ] 只拉入刚好够用的上下文，而非整个仓库
- [ ] 把模型预算花在思考是瓶颈的地方 —— [成本与效率](/zh/stack/cost)

## 复利效应

单看每个习惯都不惊艳。倍增在于它们会**叠加**：好的规划让验证变廉价；验证让审查变快；精简的上下文让模型变锐利；锐利的产出让委派变安全；安全的委派让你敢于并行推进更多。

那个循环 —— 指挥、验证、审查、重复 —— 就是全部的关键。在一个任务上把它打磨紧，然后在许多任务上跑起来。
