# Spec-Driven 开发（SDD）

[Spec-First](/zh/workflows/spec-first) 是轻量习惯：写代码前先随手写一份 spec。**Spec-Driven Development（SDD，规格驱动开发）**把这个想法推到极致 —— spec 成为*首要且持久的产物*，而代码、测试、文档都被当作从它再生的输出，由你、agent 或两者共同产生。一批开源框架把这套方法形式化了，值得了解。

## 核心观点

SDD 工具的共同信念：**在任何代码存在之前，先决定你要构建什么、以及为什么。** 如果不这样，代码库就会悄悄变成事实上的 spec —— 而一个高自主性的 agent 会飞快地、乐呵呵地把错的东西造出来。SDD 让意图变得显式、可审查、可复用。

## SDD vs TDD

它们在不同层级运作，且能很好地组合 —— SDD 不是 [TDD](/zh/workflows/tdd) 的替代品。

| | TDD | SDD |
|---|---|---|
| **首要产物** | 一个失败测试 | spec |
| **层级** | 单元 / 接口 | 功能 / 架构 |
| **范围** | 一个函数的行为 | 需求、约束、非功能性 |
| **循环** | 红 → 绿 → 重构 | spec → 规划 → 生成 → 验证 |
| **面向谁** | 开发者，紧凑循环 | 人 + agent 对齐 |

实践中：用 **SDD 决定要构建什么**，再用 **TDD 正确地把它构建出来**。spec 定验收标准；测试让每一条可执行。

## 值得知道的框架

**GitHub Spec Kit** —— 一个把 spec 置于 AI 辅助开发中心的开源工具包。你描述要构建什么，通过**结构化阶段**打磨它，然后让 agent 实现。支持众多 agent（CLI 与 IDE）。适合想要一套定义清晰、分阶段流程的场景。（[github/spec-kit](https://github.com/github/spec-kit)）

**OpenSpec** —— 一个**轻量**的 SDD 层，每个改动都有自己的文件夹 —— 提案、specs、设计、任务清单 —— **没有僵硬的阶段门禁**；你可以随时更新任何产物。支持 30+ 工具（Claude Code、Cursor、Copilot……）。适合想要结构、又不想要重仪式感的场景。（[Fission-AI/OpenSpec](https://github.com/Fission-AI/OpenSpec)）

**Superpowers** —— 一套更完整的、**以可组合技能交付**的方法论，面向编码 agent。它宣称的价值观：测试驱动、系统化优于随意、降低复杂度、以证据代替断言。它串起一整套工作流 —— 头脑风暴、把工作拆成小任务、subagent 驱动执行、TDD、代码审查 —— 且技能会在相关时自动激活。适合想要一套有主见的端到端流程的场景。（[obra/superpowers](https://github.com/obra/superpowers)）

## 到底怎么用

1. **用 spec/提案开启一个改动**，而不是用一句"写代码"的提示。
2. **让 agent 复述并提问** —— 在实现之前消除歧义。
3. **从 spec 生成规划**，审查它，再对着它实现并[验证](/zh/guides/verification)。
4. **把 spec 当作活文档** —— 现实变了就更新它，完成后归档。

## 当它显得太啰嗦时（一定会）

坦白说，缺点就是：这些框架可能**很重**。为一个一行的修复跑完整的"头脑风暴 → spec → 规划 → subagent → 审查"循环，既累又慢。解法不是抛弃它们 —— 而是**让仪式感匹配风险**：

- **让流程匹配任务。** 大的、高风险、长生命周期或团队工作 → 完整 SDD。改个错别字或一次性脚本 → 跳过，直接改。
- **选择性调用。** 你不必跑每个阶段。任务已经明确时就跳过头脑风暴，直接给规划。
- **让 spec 保持简短、高信号。** 一屏能写完的紧凑 spec，胜过一份你和 agent 都会略读的十页文档。把生成的样板裁掉。
- **别让它给简单工作注水。** 如果框架非要给一个不需要的改动加仪式，那是个信号：降级到普通的 [spec-first](/zh/workflows/spec-first) 或 [prompting](/zh/guides/prompting)。
- **归档，而非堆积。** 完成的 spec 是参考，不是杂物 —— 把它移出活跃路径，让信号保持高。

SDD 的价值在于它**对模糊、有后果的工作强加的纪律**。代价是仪式感。在它买来安全与对齐的地方付这份代价，在它只是额外开销的地方跳过它。
