[指南](/zh/guides/)讲的是单项技能。工作流则把它们组合成可复用的"配方"，对应常见的工作类型 —— 也就是有经验的实践者一次次反复用到的套路。

## 工作流

- **[测试驱动开发](/zh/workflows/tdd)** —— 把规格变成一个失败的测试，再让 agent 使它通过。agent 格外擅长的工作流。
- **[代码审查](/zh/workflows/code-review)** —— 把 agent 当作首轮审查者，同时高效地审查 agent 自己的产出。
- **[大重构](/zh/workflows/refactoring)** —— 安全地穿过庞大、机械的改动，而不至于一次性搞坏一切。
