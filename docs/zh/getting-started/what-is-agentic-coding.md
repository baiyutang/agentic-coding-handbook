---
title: 什么是 Agentic Coding？
description: 理解从代码生成到 agent 驱动开发的转变。
---

Agentic coding 是一种构建软件的新范式。你不再是让 AI 生成一段代码、再自己拼接进项目，而是把代码库和工具的访问权交给一个 AI agent，由它自主地规划、实现、测试并迭代。

## 代码生成 vs. Agentic Coding

| 维度 | 代码生成 | Agentic Coding |
|------|----------|----------------|
| **范围** | 单个片段或函数 | 跨文件的完整功能 |
| **上下文** | 你粘贴进去的内容 | 读取你的整个代码库 |
| **工具** | 无 | 终端、文件系统、LSP、浏览器 |
| **迭代** | 你审查后再次提示 | agent 自测并修复自己的产出 |
| **工作流** | 复制粘贴 | agent 直接提交 |

## Agentic Loop（智能体循环）

其核心是一个持续的循环：

1. **理解** —— agent 阅读你的代码库、文档和需求
2. **规划** —— 设计方案，把工作拆成若干步骤
3. **实现** —— 编写代码、测试并提交
4. **验证** —— 运行测试、检查类型、审查自己的产出
5. **迭代** —— 失败时进行调试并重试

## 为什么重要

Agentic coding 让开发者的角色从_编写者_转变为_审查者与架构师_。你把更多时间花在设计决策上，更少花在敲样板代码上。agent 处理机械性的工作，而你提供方向与判断。

## 数据怎么说

这不只是炒作 —— 但也不是完全自主。Anthropic 的《2026 Agentic Coding 趋势报告》描述了一个**协作悖论**：开发者如今在大约 60% 的工作里用 AI，却报告只有一小部分任务能被_完全_委派。启示是：有效的 agentic coding 是**协作式的** —— 它回报良好的准备、清晰的提示、主动的监督和人的判断，尤其在高风险工作上。

几条值得内化的模式：

- **生产力来自产出量，而非单纯的速度。** 最大的收益体现为_交付得更多_ —— 更多功能、更多修复的 bug、更多跑过的实验 —— 而不只是把同样的活干得更快。值得注意的是，AI 辅助的工作里有相当一部分，是本来根本不会去做的事。
- **角色从实现者转向编排者。** 价值转向系统设计、拆解、质量评估和指挥 agent —— 让你的判断施加在比手写每一行更广的范围上。
- **上手时间坍缩。** 在陌生代码库里进入状态，从数周降到数小时，因为 agent 补上知识缺口，而你提供监督。

> 来源：Anthropic《2026 Agentic Coding 趋势报告》。
